## 0. Why this slice + what grounds it

TIP-69/70 built the SERVER-side proof-bound NFC + FaceMatch evidence path (server validates + binds; score is adapter-declared). TIP-71 builds the **first real CaptureAgent that produces a genuine FaceMatch score** with a commercial-use-posture engine and submits it through the existing TIP-70 contract — while keeping the TagEkyc server **biometric-free** and **engine-free**. ⚠️ TIP-71 is the **first trust-adapter boundary that handles raw biometric** — so it must lock score scale, engine-failure fail-closed, credential/secret scoping, provenance, and no-overclaim (not just "call ViewFaceCore for a score"). Grounds:
- **TIP-70 server contract is ready**: `FaceMatchEvidenceDecisionBasis` (live-face + score + threshold + chip-DG2 reference to a same-session `Passed` NFC) → server-owned `PayloadHash` → manifest/proof. The agent only needs to POPULATE + submit it.
- **Engine = ViewFaceCore/SeetaFace6** (LOCKED for the reference): .NET-native, **commercial-use posture provisionally acceptable for planning, subject to the license-evidence packet + production legal/procurement review** (BSD-2 engine + MIT wrapper); same family HN212 already uses. **The claimed SeetaFace6/model provenance (not MS1M-derived) must be PROVEN by the license-evidence packet — no artifact ships on assumption.** **No known InsightFace/ArcFace/MS1M/MS-Celeb/Glint360K/VGGFace2/CASIA-derived pretrained weights** may be used; shipped artifacts MUST be traceable in the packet. dlib = documented fallback.
- **No-raw-server is foundational + as-built** (TIP-39/55; `CaptureArtifact.VaultRef` always null). The P0 raw-biometric-protection control **relocates to the agent** (the agent is where raw face bytes exist, in-memory, to run inference).
- **HN212 live-probe proved** chip-read + `CompareFace`(ViewFaceCore)=79 client-side — the concrete first agent capture path exists.

## 1. Intent

Build a **PcAgent / CaptureAgent reference implementation (.NET)** that: captures a live selfie, runs **ViewFaceCore/SeetaFace6 face-match** locally against the chip-DG2 reference, and submits a `FaceMatchEvidenceDecisionBasis` (plus its prerequisite NFC evidence) through the **existing TIP-69/70 server contracts** — with **no raw biometric persisted/logged anywhere**, deterministic non-PII tests, and a recorded license-evidence packet. **The TagEkyc server is unchanged** (proof authority only); **no server biometric ingestion, no server engine, no ONNX/ViewFaceCore dependency in any server project.**

## 1.5 Trust + raw-biometric posture (LOCKED — paste from the retention reconsideration)

> Server-side non-retention of raw biometric artifacts is an intentional privacy and liability posture, not a cryptographic proof gap. The durable signed proof binds the server-normalized decision-basis — method, engine/version, threshold, score, liveness grade, capture provenance, session/challenge binding, trusted chip-DG2 reference, PayloadHash, and manifest hash. Because TIP-69/70 compute PayloadHash server-side over that basis, the declared decision is tamper-evident within the proof chain. What this deliberately does not provide is later independent raw-image/video replay, model re-run, or third-party raw forensic audit; that capability is deferred to an optional encrypted short-TTL EvidenceVault/raw-evidence packet mode governed by legal retention policy. **Agent-side inference is execution placement, not a trust boundary.** Raw-biometric protection is relocated to the agent boundary: the agent must process raw face/liveness media in memory only, with no persistence or logging and secure disposal after inference.

Trust remains anchored server-side: the server validates the authorized agent + session/challenge/capture binding + the trusted chip-DG2 reference (same-session NFC `Passed`) + score/threshold/method/engine semantics, normalizes the result, computes the server-owned `PayloadHash`, and binds the decision-basis into the manifest/proof. The agent's score is **declared** (same trust tier as NFC's declared flags) — the agent's location/computation is NOT a trust boundary.

## 1.6 No-overclaim — FaceMatch ≠ liveness ≠ complete eKYC (round-2)

TIP-71 produces **production-grade FaceMatch evidence ONLY** (live face is similar to the trusted chip-DG2 reference). It MUST NOT claim **liveness/PAD, anti-spoofing, or complete high-assurance eKYC by itself**. FaceMatch confirms *similarity to the chip holder's photo*; liveness confirms *a live person is present* — the two do NOT substitute for each other. If a session profile requires `Liveness`, a TIP-71 FaceMatch alone MUST NOT make the session complete/high-assurance — **the server already enforces this** (`CalculateAssuranceLevel` returns High only when `DocumentNfc + FaceMatch + Liveness` are all required-and-Passed), and TIP-71 must NOT word the proof/docs to imply otherwise. Full FaceMatch + Liveness medical-pilot readiness is deferred to TIP-72.

## 2. Scope — ALLOWED

1. **CaptureAgent reference project (.NET 8) — LOCATION LOCKED (round-1).** Create `TagEkyc.CaptureAgent` as a **SEPARATE sibling solution/repo OUTSIDE `TagEkyc.sln`** (NOT a project inside the server solution; NOT in server CI). It MAY reference the TagEkyc **Contracts DTOs** through a narrow contract package/project reference, but MUST NOT add ViewFaceCore/native dependencies to any TagEkyc server project or to `TagEkyc.sln`. Captures a chip-DG2 reference (from an NFC read) + a live selfie; pluggable capture so it is testable WITHOUT a device (a fixture/file capture provider; the HN212 adapter as one real impl — §2.2). **`fixture`/`file` is an AGENT-INTERNAL capture-provider abstraction only — NOT a server `CaptureSource` enum value** (the enum has no Fixture/File). When the agent SUBMITS to the server, the artifact `CaptureSource` MUST be a **trusted** value (`PcAgent` for this reference agent, or another trusted source — `MobileSdk`/`KioskAgent`/`DeviceGateway`/`InternalAdapter`), NEVER `ExternalPreStaged` for a production-grade happy path (TIP-70 forces `ExternalPreStaged` → `ReviewRequired`). `ExternalPreStaged` is used ONLY in the negative/untrusted-capture test.
2. **ViewFaceCore/SeetaFace6 FaceMatch — the reference agent's score source.** Local 1:1 face-match producing `matchScore` (chip-DG2 vs live selfie). `engineName/engineVersion` are set on the **top-level** evidence request and become proof-bound through server normalization (e.g. `viewfacecore-seetaface6/<ver>`); basis engine fields are audit-only if used (§2.5a). The native lib is referenced ONLY by the agent project. **HN212 SDK `CompareFace` = the first chip-read/capture ADAPTER, NOT a substitute for the engine** — the FaceMatch score source for the reference agent is ViewFaceCore/SeetaFace6 UNLESS an adapter-specific HN212 score source is explicitly marked equivalent and recorded in `engineName/engineVersion`. Do not silently swap HN212 CompareFace for the ViewFaceCore engine.
3. **Submit via the EXISTING TIP-69/70 contracts (server unchanged).** The agent posts (a) the `NfcReadArtifact` capture-artifact + `NfcValidation` evidence with the chip-trust decomposition (so it reaches `Passed`), then (b) the `SelfieImage` capture-artifact + `FaceMatch` evidence with a `FaceMatchEvidenceDecisionBasis`. No new server endpoints; no server change (§3).
   - **Reference mapping (LOCKED — round-1, matches TIP-70 `ValidateFaceMatchReference`):** the DG2 face image bytes are **in-memory input to ViewFaceCore ONLY** — never persisted, never hashed into the basis, no DG2 sub-hash. The basis reference fields map to the **NFC artifact/evidence**, NOT the DG2 image:
     | basis field | value |
     | --- | --- |
     | `referenceFaceSource` | `ChipDg2FromTrustedNfc` |
     | `referenceEvidenceResultId` | id of the same-session `Passed` `NfcValidation` evidence |
     | `referenceEvidenceType` | `NfcValidation` |
     | `referenceArtifactId` | id of the `NfcReadArtifact` |
     | `referenceArtifactHash` | hash of the **`NfcReadArtifact`** (NOT the DG2 image hash) |
     | `referencePayloadHash` | the NFC evidence's `PayloadHash` |
   - **Live-face binding:** the `SelfieImage` artifact is session/challenge-bound; the basis `liveCaptureBinding` (challengeHash/sessionId/captureAgentId/deviceId/artifactHash) populates so the server stamps `CAPTURE_BOUND_TO_SESSION`.
   - **`challengeHash` algorithm PINNED (LOCKED — round-4 P1, interop contract).** The agent MUST compute it byte-identical to the server's private `BuildSessionChallengeHash` (`VerificationEvidenceApplicationService.cs:498`):
     ```
     challengeHash = "sha256:" + lowerhex( SHA256( "tip-69-capture-session-challenge" + "\n" + JCS({ "challenge": <raw session.Challenge>, "sessionId": <session.Id "N"-format guid, lowercase, no dashes> }) ) )
     ```
     where `JCS(...)` = RFC 8785 canonical JSON (keys sorted: `challenge` before `sessionId`; UTF-8; standard string escaping). The brief/`lld_03` carries a **golden test vector** (a fixed sessionId + challenge → the exact `challengeHash`) so the agent's independent implementation is proven byte-identical. **Impl (round-4):** the agent implements its OWN contract-safe canonicalizer (independent cross-verification, TIP-65 pattern) — it MUST NOT reference `TagEkyc.Application`. *(Alternative, STOP-flagged per §3: a tiny behavior-preserving move of the challenge-hash canonicalizer into a contract-safe shared location is a legitimate "truly-required" server adjustment if the build judges independent re-impl too drift-prone — flag it for review, golden byte-identical; default = independent re-impl.)*
4. **Credential / role model (LOCKED — round-1, matches as-built `ApplicationAuthorization`).** As-built separates duties: capture-artifact append requires caller category **`CaptureAgent`**; evidence append requires **`TrustedAdapter`**. The reference agent therefore uses **two logical credentials** — `CaptureAgent` for the capture-artifacts, `TrustedAdapter` for the NFC + FaceMatch evidence. (In production these MAY be separate principals: the capture agent posts artifacts; a trusted-adapter component asserts the evidence.) A test must prove a wrong-category call FAILS and the correct category passes.
   - **Secret handling (round-2 P1):** NO hardcoded API keys/secrets in repo / config / logs / test-fixtures (sample config = placeholders only); production agent credentials in an OS/user/machine secure store or deployment secret mechanism; credentials tenant/app/device-scoped where the server supports it. The agent cannot assert an arbitrary `captureAgentId`/`deviceId` — the server derives the proof-bound `captureAgentId`/`deviceId` from the **actual artifact**, not the adapter's claim.
   - ⚠️ **Cross-principal binding gap → DEBT (honest, §6):** the as-built does NOT bind the `TrustedAdapter` principal asserting the evidence to the `CaptureAgent` principal that captured the artifact (different credential categories, same session). TIP-71 does NOT pretend this is enforced; it records the gap as debt (a future server slice may bind adapter↔capture-agent↔device).
5. **Score normalization (LOCKED — round-2 P1).** The contract `matchScore` is **decimal `[0.0, 1.0]` similarity** (higher = more similar). The agent MUST normalize any engine-specific output into that scale + direction BEFORE submission: e.g. a ViewFaceCore/HN212 percent-like `79` → `0.79` (NOT `79`); a cosine **distance** (lower=better) MUST be converted to a similarity, not sent raw. **Fail-closed** on NaN / Infinity / negative / `>1` after normalization / missing score / unknown-or-unconfigured score scale (do NOT submit, or submit non-Passed — §2.6). The agent records its `normalizationRule` + raw engine score-scale in its operational diagnostics (non-proof-bound) + encodes the engine identity in the proof-bound `engineName/engineVersion` (§2.5a). (Note: the server rejects `<0`/`>1` (`FACE_MATCH_DECISION_BASIS_MISMATCH`) but CANNOT catch a same-range semantic/direction error — so correct normalization is the agent's responsibility.) Tests: `79→0.79`, `0.79` unchanged, `1.2`/`-0.1` rejected, NaN/Infinity rejected, distance-vs-similarity direction.
   - **5a. `engineName/engineVersion` source-of-truth = TOP-LEVEL request field (round-2, as-built):** the server takes `EngineName`/`EngineVersion` from the **top-level `EvidenceResultSubmissionRequestDto`** (required) and binds those into the proof-bound normalized basis (`VerificationEvidenceApplicationService.cs` uses `request.EngineName/Version`), NOT the basis's own engine fields. So the agent MUST set the **top-level** `EngineName/EngineVersion = viewfacecore-seetaface6/<ver>` (optionally `+<reader-sdk-ver>` for provenance, §2.9); the basis engine fields may be null/mirrored for audit only. A test asserts no silent top-level/basis engine mismatch.
   - **5b. Server owns threshold/isMatch:** the agent does NOT choose the threshold; `ThresholdApplied`/`IsMatch` are **omitted/null**, or if sent MUST equal the server rule (`isMatch == score >= 0.80`) else `FACE_MATCH_DECISION_BASIS_MISMATCH`. Server owns `thresholdApplied`, `isMatch`, `serverDecisionResult`, `PayloadHash`.
6. **Engine-failure fail-closed (LOCKED — round-2 P1, core eKYC).** On ANY face extraction/inference failure — **no face detected, multiple faces, poor quality / unusable crop / too small / blurred, chip-DG2 decode failure, engine exception / timeout, unsupported media format** — the agent MUST fail closed: **either do NOT submit FaceMatch evidence, OR submit it as `ReviewRequired`/`TechnicalError`-equivalent with sanitized reason codes.** The agent MUST **never** submit `Passed` when reference or live face extraction is incomplete/ambiguous (no "default score 0", no "skip-check-but-submit-Passed"). Tests: no-face, multi-face, engine-error/timeout, DG2-decode-fail, unsupported-media.
6. **No-raw-agent controls (P0 — the security crux, §4 DoD).** Raw face/frames in-memory only; no disk persistence; no logging; not in crash/debug dumps; secure disposal after inference; no real faces in repo/tests. (Acceptance is measurable but **best-effort in managed .NET** — see §4.)
7. **Deterministic non-PII tests** — the agent's score→basis mapping + disposal + credential-category logic tested with **committed embedding vectors / synthetic fixtures** (no real biometric, no device); a separate OPT-IN inference test may run ViewFaceCore on **SFHQ** (MIT synthetic faces) with tolerance (not exact).
8. **Reference-extraction provenance (round-2 P1-05 — refined, NO server change).** Audit should know WHICH reader/adapter/engine produced the evidence. ⚠️ **Pushback/refinement:** the TIP-70 `FaceMatchEvidenceDecisionBasis` has **no open `Extension` slot that the server canonicalizes into `PayloadHash`** — adding a proof-bound provenance field would be a **server change** (forbidden by the §3 no-server-change lock). So TIP-71:
   - **Proof-bound provenance NOW = the existing `engineName/engineVersion`** (top-level, §2.5a) — encode the engine + reader-SDK identity/version there (e.g. `viewfacecore-seetaface6/<ver>+hn212sdk/<ver>`). That IS in the hash graph.
   - **Full structured provenance** (chip-reader name/version, NFC-SDK version, DG2-extraction-method id, model-artifact hash) = the agent's **operational diagnostics + the license-evidence packet (non-proof-bound, disclosed as such)** — NOT written somewhere and claimed proof-bound.
   - **A richer PROOF-BOUND provenance/extension field = a SEPARATE future TIP-70 server follow-on (debt, §6)**, not TIP-71. No raw DG2 bytes / no DG2 hash / no image bytes anywhere.
9. **License-evidence packet (apply-ready — round-1).** Create a `license_evidence/` directory/file recording: exact ViewFaceCore package version; exact SeetaFace6/native model artifact filenames; **hashes of the native/model artifacts the agent ships**; copied LICENSE texts (ViewFaceCore MIT + SeetaFace6 BSD-2) or immutable links + retrieval date; the README commercial-use grant excerpt + retrieval date; the SFHQ test-dataset license/version/hash; an explicit **exclusion list** (NO InsightFace/ArcFace/MS1M/MS-Celeb/Glint360K/VGGFace2/CASIA-derived pretrained weights); and a production caveat (legal/procurement review still required before production ship). **STOP: if a model/native artifact cannot be traced to the license packet, the agent must not ship it.**

## 4. Definition of Done

**Agent function:**
- [ ] The reference agent captures (fixture or device) a chip-DG2 + live selfie, runs ViewFaceCore face-match, and submits NFC `Passed` + FaceMatch evidence such that the TagEkyc server (unchanged) validates it to a production-grade `Passed` FaceMatch bound into the proof — proven by an integration test against the real server contract.
- [ ] Top-level `EngineName/EngineVersion` + `matchScore` are submitted such that the server-normalized basis binds them (basis engine fields null/mirrored audit-only); `referenceArtifactHash` = the `NfcReadArtifact` hash (NOT a DG2 hash); `referenceFaceSource=ChipDg2FromTrustedNfc` references the same-session `Passed` NFC evidence; live capture-binding is session/challenge-bound.
- [ ] **`challengeHash` interop (P1):** the agent's independent `challengeHash` computation matches the pinned golden vector (§2.3) byte-identical; the happy-path live-binding actually reaches `CAPTURE_BOUND_TO_SESSION` against the real server. The agent does NOT reference `TagEkyc.Application`.
- [ ] **Capture-source:** the happy-path submission uses a trusted `CaptureSource` (`PcAgent`), NOT `ExternalPreStaged`; `ExternalPreStaged` appears only in the negative test (→ `ReviewRequired`).
- [ ] **Credential/role:** a test proves a `CaptureAgent`-only credential CANNOT submit evidence and a `TrustedAdapter`-only credential CANNOT submit capture-artifacts (wrong-category fails); the two-credential reference flow passes. No hardcoded secrets (placeholders in sample config).
- [ ] **Score normalization (P1):** tests for `79→0.79`, `0.79` unchanged, `1.2`/`-0.1` rejected, NaN/Infinity rejected, and similarity-vs-distance direction. `engineName/version` set at the TOP-LEVEL request field; a test asserts no silent top-level/basis engine mismatch.
- [ ] **Engine-failure fail-closed (P1):** tests prove no-face / multi-face / engine-error-or-timeout / DG2-decode-fail / unsupported-media → the agent NEVER submits `Passed` (submits nothing or `ReviewRequired`/`TechnicalError`-equivalent with sanitized reason codes).
- [ ] **No-overclaim (P1):** the agent/proof wording does NOT imply liveness/PAD or complete high-assurance eKYC; a profile requiring `Liveness` is NOT completed/High by FaceMatch alone (server-enforced; assert it).

**No-raw-agent controls (P0 acceptance criteria — measurable, best-effort in managed .NET):**
- [ ] **Best-effort caveat (honesty):** secure disposal in managed .NET = clear owned buffers, dispose images/bitmaps/Mats, avoid serialization/logging, disable/avoid dump capture in supported deployments. Do NOT claim mathematically-guaranteed memory erasure.
- [ ] **Log-sink scan:** a test runs the capture→inference→submit path and asserts NO raw biometric bytes/base64 appear in any log sink (only hashes/scores/ids).
- [ ] **Temp/disk sandbox:** a test runs the path inside a monitored temp dir (controlled working / temp / model-cache dirs where supported) and asserts the agent writes NO raw face/frame bytes to disk. **Native-lib cache/temp behavior** (ViewFaceCore/SeetaFace may write model/temp/cache outside the monitored dir) MUST be documented + either disabled, redirected into the sandbox, or proven not to contain raw biometric bytes.
- [ ] **Raw-buffer lifecycle:** documented zeroing/disposal policy for raw face/frame buffers after inference, with a test exercising the disposal path.
- [ ] **Crash/dump policy:** documented (sensitive buffers excluded from serialized state; dump capture disabled/avoided where the deployment supports it).
- [ ] Repo + tests contain **no real face images / biometric** — only deterministic embedding vectors and/or SFHQ synthetic fixtures.

**Server unchanged:**
- [ ] `dotnet test TagEkyc.sln` (server) remains green with NO server source change (or, if a tiny contract/doc adjustment was truly required and review-approved, legacy golden still byte-identical + documented).
- [ ] No ViewFaceCore/ONNX reference in any server `.csproj` (grep-verified).
- [ ] **Contract-version pin:** the agent pins the TagEkyc contract baseline (`d6e1f9b`+ TIP-70) / a recorded compatible API-contract snapshot; the integration test runs against it; a Contracts DTO change makes the agent build/test fail visibly (no silent drift).

**Compliance:**
- [ ] License-evidence packet committed (apply-ready per §2.8): exact ViewFaceCore version + SeetaFace6/native model artifact filenames + **artifact hashes** + copied LICENSE texts/immutable-links + retrieval dates + README commercial-use grant excerpt + SFHQ dataset license/version/hash + exclusion list (no InsightFace/ArcFace/MS1M-derived weights) + production caveat. Any untraceable model/native artifact → NOT shipped.

(g) is agent inference framed as placement-not-trust-boundary?
