## 0. Why this slice + what grounds it

TIP-69 (NFC) + TIP-70 (FaceMatch) made those evidence types proof-bound (server-owned PayloadHash over a canonical decision-basis). **Liveness is still on the legacy adapter-supplied path** (like FaceMatch was pre-TIP-70 — confirm in code during build: `EvidenceResultType.Liveness` + `CaptureArtifactType.LivenessMedia` + `RequiredCheckType.Liveness` exist, but no `LivenessEvidenceDecisionBasis`/server-owned hash/fail-closed). This slice gives Liveness the SAME proof-bound, server-authoritative, fail-closed treatment as FaceMatch — with the **honest passive-OSS grade** recorded so the proof never implies certified PAD. No real engine (Silent-Face = TIP-73); the liveness **score + grade + `adapterRequestedVerdict`(audit) are the agent's declared output**, while the **production verdict (`serverDerivedIsLive`) is server-derived from the score** (§2.3) — same trust tier as FaceMatch's declared score.

## 1. Intent

Make **Liveness evidence proof-bound + capture-trusted + server-authoritative + honestly-graded** by putting the liveness score + grade + capture-binding (+ the server-derived `isLive`) into a canonical **`LivenessEvidenceDecisionBasis`** whose digest is the server-owned `PayloadHash` (`canonical basis → PayloadHash → manifest → proof`). **No manifest/proof SHAPE change; legacy golden byte-identical.** The liveness check binds to the SAME session-bound live capture as FaceMatch (the live selfie/video frames). No real Silent-Face engine here (TIP-73); no liveness inference on the server.

## 1.5 Honest grade framing (LOCKED — the liveness-specific crux)

- **Passive-OSS liveness is NOT certified PAD.** The basis records `method` + `livenessGrade` (`passive-2d-only`, NOT `certified-iso30107`). The proof MUST NOT imply certified anti-spoofing. Per strategy: passive OSS stops only basic 2D attacks (print/screen); deepfake / video-injection / 3D-mask are NOT covered.
- **Earned-method invariant (round-1 P1, mirror TIP-71 earned-engineName):** the fixture/declared path in THIS slice uses a non-production `method=fixture-liveness` label. The `silent-face` / `minifasnet` method name is RESERVED for a REAL engine call (TIP-73) with pinned model/license/hash provenance — TIP-72 must NOT stamp `silent-face` onto a verdict no real Silent-Face produced (no unearned engine/method). `engineName/engineVersion` follow the SAME rule + are top-level-only (§2.5a).
- **Liveness ≠ FaceMatch ≠ identity.** Liveness attests "a live person was present"; it does NOT attest WHO (that's FaceMatch + the chip-DG2 reference). The two are distinct required-checks.
- **Server-authoritative over the RESULT, not the raw judgment:** the agent declares the liveness verdict/score; the server enforces the grade/threshold/binding + may coerce `ReviewRequired`, but does NOT re-run a PAD engine (TIP-73+). "Liveness passed" ≠ "server independently verified liveness".
- ⚠️ **Known assurance-honesty gap (carried debt) + the explicit production caveat (round-1 P1):** as-built `CalculateAssuranceLevel` returns High when `DocumentNfc + FaceMatch + Liveness` are all required-and-Passed — so a **passive-OSS** Liveness=Passed contributes to High the same as a certified PAD would, AND (no consumer-visible substate) a relying party sees `High` without seeing "passive-OSS, not certified." TIP-72 records the grade honestly in the basis but does NOT change `CalculateAssuranceLevel` or add a consumer surface (both = assurance-policy-catalog + consumer-substate-resolver follow-ons). **EXPLICIT CAVEAT:** **TIP-72 (passive-OSS liveness) is NOT sufficient for any production claim that depends on certified / strong PAD** — `High` reached via passive-OSS liveness is an integrity-tier signal, NOT a certified-liveness assurance. This is accepted debt, not a silent overclaim; the gap is closed only when the policy catalog grades passive-vs-certified into assurance.

## 2. Scope — ALLOWED (mirror TIP-70)

1. **`LivenessEvidenceDecisionBasis` (subject-data-classified like TIP-70)** — additive field on the Liveness evidence submission: `{ liveMediaArtifactId + liveMediaArtifactHash; livenessScore (decimal, normalized [0,1]); adapterRequestedVerdict (live|spoof|uncertain — AUDIT/cross-check only); serverDerivedIsLive (= score ≥ threshold, server-set); method (`fixture-liveness` for the fixture path; `silent-face`/`minifasnet` RESERVED for a real engine, §1.5); livenessGrade (passive-2d-only); thresholdApplied (server); capture-binding obj (challengeHash domain-separated, sessionId, captureAgentId, deviceId, capturedAt, artifactHash) — for the LIVE media; serverDecisionResult; adapterRequestedResult (audit); sanitizedSummaryLabel }`. **`engineName/engineVersion` are NOT in the basis — top-level request fields ONLY, proof-bound via `evidenceEngines`/manifest (§2.5a, mirror TIP-70); no split-brain (top-level vs basis must never disagree).** Classification: **no raw subject biometric / video / frames** — only hashes + ids; pseudonymous biometric-derived hashes + operational ids = internal-confidential, not consumer-visible.
2. **Server-OWNED `PayloadHash`** — server canonicalizes the FINAL normalized basis + computes `PayloadHash` via `EvidenceCanonicalization.HashCanonical("tip-72-liveness-decision-basis", basis)`. Adapter-supplied `PayloadHash` ignored / if present must match else reject `LIVENESS_PAYLOAD_HASH_MISMATCH`. (`engineName/version` = TOP-LEVEL request field, proof-bound — basis fields audit-only; mirror TIP-70 §2.5a.)
3. **Server-DERIVED verdict + fail-closed validator (round-1 P2 — Option B, mirror FaceMatch `isMatch`):** the server DERIVES the production `serverDerivedIsLive` from `score ≥ configuredThreshold` (NOT the adapter's verdict). For `Liveness=Passed`, require `serverDerivedIsLive == true` AND a positive `CAPTURE_BOUND_TO_SESSION` on the live media. The **`adapterRequestedVerdict` is AUDIT/cross-check only** — if it conflicts with the server-derived result (e.g. adapter says `live` but score < threshold, or `spoof` but score ≥ threshold), record `LIVENESS_VERDICT_MISMATCH` and coerce `ReviewRequired`. Below-threshold → `ReviewRequired` + `LIVENESS_BELOW_THRESHOLD`; missing basis → `LIVENESS_DECISION_BASIS_REQUIRED`; untrusted/negative binding → `ReviewRequired` + `DIRECT_CLIENT_UPLOAD_UNTRUSTED`; adapter `PayloadHash` mismatch → `LIVENESS_PAYLOAD_HASH_MISMATCH`. `serverDecisionResult` normalized BEFORE hashing (payload ≡ persisted Result). Threshold = server config. Score is adapter-declared (engine = TIP-73).
4. **Live-media artifact rule LOCKED (round-1 P1-02) + role-aware binding:** preferred source = a dedicated `CaptureArtifactType.LivenessMedia` when supplied; fallback = the session-bound live `SelfieImage` used by FaceMatch, allowed ONLY if it is the EXACT live capture artifact and `CAPTURE_BOUND_TO_SESSION` is server-trusted. `ExternalPreStaged` / any un-bound artifact is NEVER production-grade Passed. If video/frames are used, the server stores ONLY the artifact id/hash — NEVER raw media. `CAPTURE_BOUND_TO_SESSION` applies to this live-media artifact; server-stamped iff `IsCaptureBindingTrusted`.
5. **Honest grade recording** — `method` + `livenessGrade` (passive-2d-only) live in the proof-bound `LivenessEvidenceDecisionBasis` (hashed into `PayloadHash`); **`engineName/engineVersion` stay TOP-LEVEL + proof-bound via `evidenceEngines`/manifest — NOT duplicated in the basis** (§2.1). `verified: passed` must never imply certified PAD / ISO-30107. `CalculateAssuranceLevel` **unchanged** (grade→assurance = follow-on; §1.5 gap flagged).
6. Tests (deterministic, **no raw biometric/video** — synthetic hashes + numeric scores/verdicts) + docs (lld_03 additive Liveness basis + error codes).

## 3. STOP / NOT in this slice (ceiling)

- **NO real Silent-Face / PAD engine, NO ONNX, NO image/video decode, NO liveness inference on the server** — score adapter-declared; the production verdict is server-derived (§2.3); the real engine = TIP-73 (agent-side).
- **`fixture-liveness` method/engine is TEST / local-dev ONLY (round-2):** a production-grade `Passed` MUST NOT rely on a `fixture-liveness` method/engine (mirror TIP-71 earned-engine). `silent-face`/`minifasnet` is stamped ONLY by a real engine call (TIP-73).
- **NO manifest/proof SHAPE change** — bind via the existing `PayloadHash`; no new `ManifestEvidenceRefDto`/`EvidenceProofEngineRef`/decision-seed fields.
- **NO `CalculateAssuranceLevel` change** (grade→assurance = assurance-policy-catalog follow-on; the passive-OSS→High honesty gap is flagged, not fixed here).
- **NO certified-PAD / ISO-30107 / anti-deepfake / liveness-active-challenge claim** — passive-OSS basic-2D only, recorded honestly.
- **NO consumer-visible substate surface** (no PayloadHash/score/grade to BusinessConsumer).
- **NO enum change** (`EvidenceResultType.Liveness` etc. already exist; no new members).
- **NO raw subject biometric/video** anywhere; legacy TIP-65/66 golden byte-identical; TIP-69 NFC + TIP-70 FaceMatch paths untouched.

## 4. Definition of Done (mirror TIP-70)
